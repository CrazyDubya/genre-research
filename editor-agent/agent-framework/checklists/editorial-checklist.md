# Editorial Checklist

Comprehensive checklists for each editorial pass and final review.

---

## PRE-EDITING CHECKLIST

### Before Starting Any Edit

- [ ] Manuscript received and accessible
- [ ] Format is workable
- [ ] Word count verified
- [ ] Genre/category confirmed
- [ ] Special requirements known
- [ ] Timeline established
- [ ] Previous feedback available (if any)
- [ ] Style guide requirements known
- [ ] Author preferences documented (if any)

### Preparation Complete

- [ ] Workspace set up
- [ ] Reference materials ready
- [ ] Tracking method chosen
- [ ] Style sheet template ready
- [ ] Time allocated appropriately

---

## DEVELOPMENTAL PASS CHECKLIST

### First Read (Reading Without Editing)

- [ ] Read complete manuscript without heavy marking
- [ ] Note emotional reactions and engagement
- [ ] Track where confused or disengaged
- [ ] Observe overall impressions
- [ ] Resist urge to fix prose issues

### Structural Review

**Act One**
- [ ] Setup establishes world and character
- [ ] Inciting incident present and effective
- [ ] Stakes introduced
- [ ] First plot point drives into Act Two
- [ ] Pacing appropriate

**Act Two**
- [ ] Conflict escalates appropriately
- [ ] Midpoint shift/revelation present
- [ ] Character development continues
- [ ] Subplots advance
- [ ] Stakes increase
- [ ] Crisis/dark moment effective

**Act Three**
- [ ] Climax positioned correctly
- [ ] Resolution satisfies story promises
- [ ] Character arcs complete
- [ ] Themes crystallize
- [ ] Pacing appropriate

### Plot Review

- [ ] Central conflict clear and sustained
- [ ] Cause-and-effect logic sound
- [ ] No significant plot holes
- [ ] All major threads resolved
- [ ] Stakes appropriate and escalating
- [ ] Ending earned

### Character Review

- [ ] Protagonist compelling and motivated
- [ ] Character arc present and effective
- [ ] Supporting characters serve story
- [ ] Antagonist effectively developed
- [ ] Relationships develop believably
- [ ] Character consistency maintained

### Pacing Review

- [ ] Opening hooks effectively
- [ ] Momentum maintained through middle
- [ ] Tension varies appropriately
- [ ] No significant sag points
- [ ] Ending paced well

### Developmental Feedback

- [ ] Assessment organized by priority
- [ ] Specific examples provided
- [ ] Recommendations are actionable
- [ ] Strengths acknowledged
- [ ] Letter/document prepared

---

## LINE EDITING CHECKLIST

### Pre-Line Edit

- [ ] Developmental issues resolved
- [ ] Structure finalized
- [ ] Voice baseline established
- [ ] Content is staying (worth polishing)

### Voice Check

- [ ] Voice characteristics documented
- [ ] Voice maintained throughout
- [ ] Intentional variations identified
- [ ] Changes preserve author's voice

### Sentence Level

- [ ] Clarity achieved
- [ ] Concision appropriate
- [ ] Sentence variety present
- [ ] Rhythm serves content
- [ ] Awkward constructions addressed

### Paragraph Level

- [ ] Paragraphs have clear purpose
- [ ] Appropriate length variety
- [ ] Strong openings
- [ ] Effective transitions
- [ ] Flow between paragraphs

### Word Level

- [ ] Word choice precise
- [ ] Unintentional repetition addressed
- [ ] Register consistent
- [ ] Clich√©s minimized
- [ ] Jargon appropriate

### Dialogue

- [ ] Dialogue sounds natural
- [ ] Character voices distinct
- [ ] Tags effective and unobtrusive
- [ ] Subtext present where appropriate
- [ ] Balance with action/narrative

### Description

- [ ] Description serves story
- [ ] Specific details used
- [ ] Multiple senses engaged
- [ ] Integrated with action
- [ ] Appropriate to POV

### Line Edit Complete

- [ ] All chapters reviewed
- [ ] Patterns documented
- [ ] Voice preserved throughout
- [ ] Queries organized
- [ ] Ready for next pass

- [ ] Line editing complete

- [ ] Content finalized

- [ ] Subject-verb agreement
- [ ] Pronoun agreement and reference
- [ ] Tense consistency
- [ ] Parallel structure
- [ ] Modifier placement
- [ ] Sentence completeness

### Punctuation Check

- [ ] Comma usage correct
- [ ] Dialogue punctuation correct
- [ ] Semicolons used appropriately
- [ ] Em/en dashes correct
