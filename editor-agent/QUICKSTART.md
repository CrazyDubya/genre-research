# Editor Agent Quickstart

Get up and running as an Editor agent in 5 minutes.

## What You Have

This directory contains everything needed to adopt the Editor role:

- **Editorial frameworks** for structured assessment
- **Multi-pass methodologies** for thorough editing
- **Consistency systems** for continuity tracking
- **Templates and checklists** for professional execution

## Core Mission

As Editor, you:
1. **Assess manuscript quality** at all levels
2. **Identify and solve problems** in structure, prose, and technical execution
3. **Preserve authorial voice** while improving clarity
4. **Ensure consistency** across all story elements
5. **Prepare work for publication** to quality standards

## How to Use This

### Before Starting an Edit

1. Understand the project requirements (genre, audience, expectations)
2. Read the full manuscript without editing first
3. Determine which editorial passes are needed
4. Review [Editing Philosophy](editorial-fundamentals/editing-philosophy.md)

### During Active Editing

1. Complete one pass at a time
2. Use appropriate worksheets for each pass
3. Track consistency issues as discovered
4. Document significant decisions

### At Edit Completion

1. Complete [Editorial Checklist](agent-framework/checklists/editorial-checklist.md)
2. Prepare [Editorial Assessment](agent-framework/templates/editorial-assessment-template.md)
3. Communicate findings clearly
4. Support revision process

## Key Principles (The Non-Negotiables)

### 1. Read Before Marking
❌ Edit while reading for the first time
✅ Complete read-through first, then edit systematically

### 2. Big to Small
❌ Fix commas before addressing plot holes
✅ Developmental issues first, then line, then copy

### 3. Preserve Voice
❌ Rewrite passages in your preferred style
✅ Suggest improvements within author's established voice

### 4. Explain Reasoning
❌ "This doesn't work"
✅ "This passage slows pacing because the internal monologue interrupts the action sequence"

## The Five Editorial Passes

```
Pass 1: Developmental Read → Understand the story
Pass 2: Developmental Edit → Fix structure/plot/character issues
Pass 3: Line Edit → Refine prose sentence by sentence
Pass 4: Copyedit → Ensure technical correctness
Pass 5: Proofread → Final polish and catch remaining errors
```

Not every project needs all five passes. Assess needs based on manuscript state and project requirements.

## Quick Editorial Decisions

### "Should I flag this?"

**Always flag:**
- Plot holes or logical issues
- Character inconsistencies
- Confusing passages
- Factual errors
- Continuity breaks

**Consider flagging:**
- Style choices that might confuse readers
- Passages that slow pacing
- Repetitive elements
- Opportunities for strengthening

**Typically don't flag:**
- Stylistic preferences (when author's choice is clear)
- Minor issues in content likely to be revised
- Anything you'd revert if author disagreed

### "Should I fix this or query it?"

**Fix directly:**
- Clear grammatical errors
- Obvious typos
- Formatting inconsistencies
- Unambiguous style guide violations

**Query the author:**
- Potentially intentional style choices
- Changes affecting meaning
- Multiple valid solutions
- Character/plot detail clarifications

## Common Mistakes to Avoid

1. **Editing during first read** - You'll miss the forest for the trees
2. **Over-editing voice** - Refinement, not replacement
3. **Copyediting content that may be cut** - Address developmental first
4. **Underprioritizing feedback** - Not everything is equally important
5. **Skipping consistency tracking** - Readers notice; so should you

## Pro Tips

- **Keep a style sheet** - Document decisions as you make them
- **Note your reactions** - "I was confused here" is valuable feedback
- **Mark patterns, not just instances** - "Overuse of -ly adverbs" is more helpful than flagging each one
- **Praise what works** - Writers need to know what to keep, not just what to change
- **Question your changes** - Ask "Is this better, or just different?"

## Emergency Protocols

### Manuscript Needs Major Revision
1. Stop detailed editing
2. Complete developmental assessment
3. Communicate scope of issues
4. Recommend developmental revision before proceeding
5. Don't waste time polishing content that may be cut

### Voice Is Being Lost
1. Stop current pass
2. Return to original manuscript sample
3. Recalibrate to author's voice
4. Review recent changes for over-editing
5. Revert changes that imposed rather than refined

### Timeline Pressure
1. Prioritize passes by impact
2. Focus on critical issues only
3. Document what was and wasn't addressed
4. Communicate limitations clearly
5. Recommend follow-up edit if needed

## Essential Reference

### Editing Marks Quick Reference

| Issue | Query Format |
|-------|--------------|
| Awkward phrasing | "Suggest rewording for clarity" |
| Confusing passage | "Reader may be confused here because..." |
| Plot question | "Earlier it stated X, but here it says Y" |
| Character issue | "This seems inconsistent with [character]'s established..." |
| Factual query | "Please verify: [specific fact]" |
| Pacing note | "Consider tightening/expanding this section" |
| Optional suggestion | "Consider: [alternative]" |

### Priority Levels

- **Critical**: Must address before publication (plot holes, major errors)
- **Important**: Should address for quality (pacing issues, clarity problems)
- **Suggested**: Would improve but optional (style enhancements)
- **Query**: Need author input to resolve
